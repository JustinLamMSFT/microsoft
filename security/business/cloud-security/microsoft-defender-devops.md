## Defender for DevOps architecture

Security teams struggle with fragmented visibility, lack of business risk context, and integrated processes to effectively shift left. Defender for DevOps helps unify, strengthen, and manage DevOps security from development to runtime within Defender for Cloud.

.heading-bg-color-layout-container-uid647f{ background-color: !important; }

![A diagram breaking down the Defender for DevOps architecture](https://cdn-dynmedia-1.microsoft.com/is/image/microsoftcorp/DevOps%20Security%20Management%20R6?resMode=sharp2&op_usm=1.5,0.65,15,0&wid=1800&qlt=100&fmt=png-alpha&fit=constrain)

.heading-bg-color-layout-container-uidbc3b{ background-color: !important; }

[More about the diagram](https://www.microsoft.com/en-us/security/business/cloud-security/microsoft-defender-devops?rtc=1#Moreaboutdiagram)

 

Defender for DevOps empowers security teams to unify, strengthen and manage DevOps security within Defender for Cloud, from development to runtime. It provides full visibility into the DevOps inventory and the security posture of pre-production application code and resource configurations across multiple-pipeline and multicloud environments. It also strengthens cloud resource configurations throughout the development lifecycle with IaC templates and container images to help prevent cloud misconfigurations from reaching production environments. Defender for DevOps helps prioritize and streamline remediation processes by connecting application code insights with the runtime context.
